    JavaScript Weekly #580

    2022/3/12

    🤦 Thanks to all readers who noticed we made a mistake with last week's subject line "TypeScript 2.6 released" – rather than the intended 4.6. We were only 4 years late (opens new window) with that news 😉 Rest assured, more caffeine has been consumed today..

    (opens new window)

    An ECMAScript Proposal for Type Annotations in JavaScript (That Are Ignored at Runtime) (opens new window) — A huge amount of chatter this week over an interesting proposal to support type annotations directly in standard JavaScript code that are available for external type checkers to use but which are otherwise ignored at runtime. TypeScript’s Daniel Rosenwasser has written a lot more about the approach here (opens new window). Dr. Axel has also written up his thoughts. (opens new window) Whatever the case, people seem pretty excited for the potential and Allen Wirfs-Brock has said "if I was still active on TC39 I’d be advocating for this proposal." (opens new window)

    (opens new window)

    Wallaby.js: 15x Faster JavaScript/Jest Testing and Debugging (opens new window) — Wallaby runs your tests immediately as you type, with runtime values, errors and code coverage displayed in real-time right next to your code. Features have been designed specifically for testing, including: time-travel debugging, edit-and-continue support, and much more.

    React 18 Release Candidate Released (opens new window) — The full production release of React 18 is seemingly imminent: just out, the detailed documentation covering the upgrade to this week’s release candidate. The post also includes useful information about some of the major new features.

    🐦  Twitter Thread on Stripe Converting from Flow to TypeScript (opens new window) — And it took just one weekend: “We modified about 3.5 million lines of code, and then hundreds of developers came in Monday morning ready to write TS.”

    WebGPU — All of the Cores, None of the Canvas (opens new window) — It’s always a pleasure to see Surma dig into something (especially Web Workers (opens new window)) and this time he turns to making the potential of WebGPU (an API that exposes the capabilities of GPU hardware that, admittedly, isn't available by default in any standard browser yet) more accessible to us all.

    # IN BRIEF:

    • Did you know that for legacy reasons HTML-style comments work in JavaScript? (opens new window) (A quirk that adds some additional complexity to V8's parser.. (opens new window))

    • A fascinating Twitter thread diving straight down the rabbit hole of historic calendars and why JS and Swift disagree on when January 1, 1 AD actually was (opens new window).

    • Chrome has made some big leaps forward in browser performance (opens new window) in the past year and now claims to be faster than Safari.

    • The Views on Vue podcast has an episode covering ▶️ the growth of the Vite ecosystem (opens new window).

    Naming Conflicts – or How Existing Code Can Force Proposed Features to Be Renamed (opens new window) — Phil Karlton once said “cache invalidation and naming things” were the two hardest things in computer science, and JavaScript has had its fair share of trouble around the latter.

    A Lightweight GitHub REST API in 34 Lines of Code (opens new window) — The power of ES proxies strikes again. Not a tutorial, but you might learn something from the code.

    My Experience Building a Chess App in React (opens new window) — The author describes himself as a self-taught, relative newcomer to React development.     # 🛠 Code & Tools

    LemonadeJS V2: A Reactive JavaScript Library with Zero Transpiling Needed (opens new window) — Simple, straightforward reactivity anywhere with as little tooling surrounding it as possible.

    striff: Real Simple String Diffing (opens new window) — Given two strings, you can find out which characters were added or removed, and where.

    PSD: A Zero-Dependency PSD (Photoshop) Parser (opens new window) — Will parse info for each layer including text and also supports Photoshop’s .PSB (big image) format. GitHub repo (opens new window).

    Studio 3T Manages Your Data, While You Wrestle Your App into Shape (opens new window) — Studio 3T's full suite of MongoDB tools lets you develop with data even faster and turn queries into code.

    Tygo: Generate TypeScript Types from Go(lang) Source Code (opens new window) — Let’s say you’re building an app with Go on the backend and TypeScript on the frontend. This tool scans your Go-based API and creates the relevant TypeScript types for you.

    Reason: Write Fast, Type Safe Code in OCaml, But for the JS Ecosystem (opens new window) — You essentially get to write OCaml but compile to JavaScript.

    Ultra: A Modern Streaming React Framework for Deno (opens new window) — Built around native browser features like ES modules, import maps, and web streams. The source (opens new window) of the project’s own Ultra-powered homepage may help you weigh up if you like its cutting edge approach or not.
